## 🆕 Что нового в v8.0

### Капча и безопасность
- **Строгая верификация** через приватный канал с автоматической проверкой членства
- Проверка `channel_member` перед каждой функцией
- Защита от обхода капчи

### Business Chatbot
- Проверка подключения Business через `BusinessConnection` events
- Статус подключения в профиле пользователя
- Кнопка "Проверить подключение" для диагностики

### Userbot (Pyrogram)
- **Персональные API ключи** для каждого пользователя (хранятся в БД)
- 4-шаговая настройка: api_id → api_hash → телефон → код → (2FA)
- Полная инструкция получения ключей внутри бота

### Перехват сообщений
- **Сохранение медиа в BLOB** для исчезающих сообщений (TTL)
- Поддержка **view_once** сообщений («один просмотр»)
- Немедленная скачка медиа до удаления
- Массовое удаление: ZIP-архив с HTML + JSON + медиа-файлами

### Клонирование ботов
- **Полный функционал** в клонах (не заглушки)
- Каждый клон работает как основной бот
- Отслеживание пользователей клонов в `clone_users`
- Счётчик `user_count` для каждого клона

### Админ-панель
- Система конфигов `system_config` для глобальных настроек
- Упрощённая навигация по пользователям
- Просмотр статистики клонов и userbot-сессий

Открой `merai_v8_bot.py`, строки 54–62:

```python

CAPTCHA_CHAN_ID = -1003716882147                              # ← ID канала (как получить — ниже)

# Глобальные API ключи Pyrogram (опционально, можно оставить 0/"")
```

**Важно:** если `PYRO_API_ID/HASH` оставить пустыми, каждый пользователь будет вводить свои через бота.

### 3. Получение ID приватного канала

```
1. Создай приватный канал
2. Добавь бота как администратора с правом "Пригласить пользователей"
3. Включи "Одобрение заявок вручную" в настройках канала
4. Отправь в канал любое сообщение
5. Перешли сообщение боту @username_to_id_bot → получишь CHAT_ID
6. Вставь ID в CAPTCHA_CHAN_ID
7. Скопируй invite link → вставь в CAPTCHA_CHANNEL
```

## ⚙️ Настройка капчи (критично!)

Бот **НЕ ОДОБРЯЕТ** заявки автоматически. Капча работает так:

1. Пользователь нажимает "Пройти верификацию" → попадает в приватный канал
2. Нажимает "Отправить заявку на вступление"
3. Бот получает `ChatJoinRequest` → устанавливает `is_verified=1`, `channel_member=1`
4. Пользователь возвращается в бота → получает доступ

**Важно:** заявку **не нужно одобрять вручную**. Сам факт отправки заявки = прохождение капчи.

## 🔧 Архитектура БД (новое в v8.0)

### Новые поля в `users`:
- `channel_member` — состоит ли в капча-канале
- `biz_connected` — подключён ли Business chatbot
- `pyro_api_id`, `pyro_api_hash` — личные API ключи

### Новые таблицы:
- `clone_users` — пользователи клонов
- `system_config` — глобальные настройки

### Обновлено в `msg_cache`:
- `media_bytes BLOB` — сырые байты медиа
- `ttl_seconds` — для исчезающих
- `view_once` — для «один просмотр»

## 🚨 Важные напоминания

1. **Никогда не используй `input()`** — BotHost headless
2. **Все токены вшиты в код** — .env не нужен
3. **Проверь CAPTCHA_CHAN_ID** — бот не будет работать без верного ID
4. **API ключи Pyrogram** — можно оставить пустыми, пользователи введут свои
5. **Клоны запускаются в фоне** — не блокируют основной бот
